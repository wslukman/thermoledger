Introduction

Selamat datang di dokumentasi resmi pengembang **Thermoledger** (PT Universal Digital Property).

Thermoledger adalah sebuah paradigma baru dalam teknologi desentralisasi yang meninggalkan komputasi digital biner tradisional ($0$ dan $1$) yang kaku dan boros energi. Proyek ini mengimplementasikan arsitektur **Thermodynamic / Physics-as-Computation**, di mana validasi konsensus dan perpindahan nilai tidak diatur oleh kode smart contract linear, melainkan oleh pencapaian kesetimbangan entropi fisik alami.

Core Concept

Dalam ekosistem ini, koin native $DUIT dirancang secara hiper-langka (maksimal suplai: 1.000.000 koin) dengan pecahan terkecil bernilai Joule (J), di mana 1 DUIT = 10^8 Joule.

Mengapa Termodinamika?

Komputer biner saat ini menghabiskan energi yang sangat besar untuk memecahkan puzzle matematika buatan manusia (seperti SHA-256 pada Bitcoin). Renungan membalikkan paradigma ini dengan memanfaatkan hukum fisika alam sebagai penentu konsensus. Dengan menyelaraskan sirkuit analog ke keadaan energi minimum, proses validasi transaksi kami secara teoritis mendekati batas efisiensi energi Landauer (Landauer's Limit).

Core Architecture

Arsitektur Project Renungan dirancang di atas dua lapisan jaringan (Layer 1 & Layer 2) yang diamankan secara langsung oleh hukum termal.

1. Paradoks "Jendela Noise" & Keamanan Termal

Sistem kami menolak konsep keacakan semu (pseudo-random) buatan algoritma biner. Sebagai gantinya, jaringan memanfaatkan thermal noise alami dari sirkuit perangkat keras:

  • One-Time Pad Alami: Noise termal bertindak sebagai kunci kriptografi sekali pakai yang berubah setiap milidetik mengikuti fluktuasi entropi atomik perangkat keras.
  • Imunitas Peretasan: Untuk memalsukan transaksi, peretas harus memanipulasi hukum fisik entropi lokal pada sirkuit validator pada milidetik yang sama. Transaksi berbahaya otomatis teredam sebagai "distorsi kotor" oleh sirkuit analog.

2. Local Thermodynamic Ponds (Layer 2)

Di Layer 2, transaksi mikro dikumpulkan dalam bentuk **Bukti Matriks Entropi** sebelum dikunci ke Layer 1. Bot arbitrase berkompetisi membangun transaksi mikro yang saling menguntungkan (*cooperative balance*) di dalam kolam lokal tanpa risiko kehilangan biaya energi (tidak ada gas fee wars).

State Energy Proof (SEP)

State Energy Proof (SEP) menggantikan mekanisme konsensus Proof-of-Work (PoW) dan Proof-of-Stake (PoS) konvensional.

Hukum Entropi Jaringan

Konsensus diatur secara ketat oleh hukum termodinamika alam:

ΔS_universe >= 0

Setiap transaksi divalidasi dengan memetakan jalur komputasi analog menuju kesetimbangan energi minimum (**Minimum Energy Path**). Validitas dicapai ketika status sirkuit validator lokal mencapai ekuilibrium termal yang sinkron dengan protokol global global.

Time-Lock Fisik: Cooling-Off Period

Jika terjadi anomali atau gangguan internet global massal, sistem tidak memerlukan koordinasi manual manusia (*hard fork*):

  1. Node akan mendeteksi penurunan sinkronisasi getaran entropi global.
  2. Sistem otomatis masuk ke mode **Membeku (Safe Mode)** secara mandiri, mengunci seluruh status saldo $DUIT.
  3. Sistem kembali **Mencair (Thaw)** secara otomatis begitu sirkuit fisik mendeteksi stabilitas kesetimbangan energi termal jaringan telah pulih.

Kapasitas Blok: Batas Termal vs. Batas Gas

Pada blockchain biner konvensional (seperti Ethereum), kapasitas transaksi per blok dibatasi oleh Gas Limit digital (yaitu batas beban komputasi CPU). Di Renungan, kapasitas ini ditentukan oleh Kapasitas Termal Sirkuit dan Toleransi Ekuilibrium Entropi:

  • Kompresi Melalui Cooperative Balance (L2 Pond): Jika terjadi ribuan transaksi mikro bersamaan, bot arbitrase menggunakan Least Action Principle untuk mengurutkan transaksi agar saling meniadakan (offset). Hal ini meminimalkan putaran magnet fisik (spin flips) pada sirkuit L1, sehingga beban entropinya mendekati nol dan semuanya dapat diselesaikan dalam satu transisi blok L1 yang sama.
  • Batas Disipasi Termal (Thermal Stress): Jika transaksi berjalan satu arah secara masif tanpa offset, sirkuit validator akan mengalami stres magnetik tinggi yang menaikkan suhu fisik chip. Jika suhu melewati batas kritis 5.0°C, sistem akan memicu Cooling-Off Period secara alami untuk mendinginkan sirkuit sebelum menerima blok transaksi baru kembali.

Client Dart SDK

Pustaka SDK Dart disediakan untuk diintegrasikan langsung ke dalam ekosistem aplikasi kasir retail Merchant DuitLap dan dompet digital konsumen.

Integrasi Kode Dart

Contoh inisialisasi ekosistem dan eksekusi pembayaran mikro:

import 'package:duit_thermo_sdk/duit_thermo_sdk.dart';

void initializeDuitEcosystem() async {
  // 1. Menginisialisasi koneksi ke node lokal atau jembatan hardware
  DuitWallet wallet = DuitWallet(nodeUrl: "https://node1.duitscan.io");
  
  // 2. Mendeteksi dan mengunci koneksi dengan Hardwallet Dongle USB/NFC
  bool isHardwareConnected = await wallet.connectHardware();
  
  if (isHardwareConnected) {
    print("Hardwallet Analog Terdeteksi. Keamanan dijamin Hukum Termodinamika.");
  }
}

void executeMerchantPayment(String recipientAddress, double amountInDuit) async {
  // Pecahan otomatis dikonversi ke satuan internal Joule (1 DUIT = 10^8 Joule)
  int amountInJoule = DuitConverter.toJoule(amountInDuit);
  
  try {
    // Eksekusi transaksi mikro instan melalui enkapsulasi Layer 2
    var txResult = await DuitWallet.sendMicroTransaction(
      recipient: recipientAddress,
      jouleAmount: amountInJoule,
      authMode: AuthMode.biometricEntropy
    );
    
    print("Transaksi Kasir Sukses. Terkunci di Blok: \${txResult.blockHeight}");
  } catch (e) {
    print("Gagal mencapai kesetimbangan energi transaksi: \$e");
  }
}

Alur Konfirmasi & Integrasi Web3 (Connect & Approve UX)

Ekosistem aplikasi pihak ketiga (dApp) berinteraksi dengan dompet pengguna melalui alur keamanan standar industri yang sangat ketat:

  1. Wallet Handshake: Pengguna memberikan otorisasi kepada website merchant untuk membaca alamat publik wallet-nya. Website tidak memiliki akses untuk memotong saldo secara sepihak.
  2. Transaction Proposal: Ketika ada permintaan pembayaran, aplikasi web mengajukan draf transaksi ke node untuk menangkap parameter termal terupdate (Entropy Salt) melalui API /prepare.
  3. Physical Confirmation: Dompet pengguna mendeteksi pengajuan tersebut dan meminta persetujuan. Pengguna melakukan tindakan fisik autentikasi:
    • Melakukan tap kartu fisik NFC ke terminal pembaca atau HP.
    • Menyentuh sensor sidik jari atau tombol fisik pada USB Dongle yang terpasang.
    • Melakukan verifikasi PIN/FaceID pada aplikasi Hot Wallet handphone.
  4. Physic Sign & Commit: Modul sirkuit hardware menandatangani bukti energi State Energy Proof (SEP) dan mengirimkannya ke validator via API /commit untuk dicatat permanen dalam blok L1.

Model Keamanan Hardware & Mitigasi Pencurian (Threat Model)

Ekosistem dirancang untuk memitigasi skenario kehilangan atau pencurian perangkat keras fisik (NFC Card atau USB Dongle) melalui skema keamanan berlapis:

  • Batas Transaksi Tanpa PIN (Velocity Limits): Transaksi ritel cepat (tap-and-go) dibatasi hanya untuk nilai nominal kecil. Transaksi di atas ambang batas (threshold) mewajibkan input PIN 6-digit pada layar mesin pembaca yang divalidasi secara langsung di dalam chip aman (Secure Element) kartu.
  • 2-of-2 Multi-Signature (NFC + Phone App): Konfigurasi dompet tingkat lanjut membagi kunci privat menjadi dua share terpisah. Tanda tangan konsensus SEP hanya sah jika kartu NFC di-tap DAN aplikasi mobile menyetujui transaksi melalui FaceID/Sidik Jari HP pengguna.
  • Active Biometrics pada USB Dongle: Khusus perangkat USB Dongle, otentikasi menggunakan sensor sidik jari fisik di atas dongle. Pencurian fisik tanpa sidik jari yang cocok membuat perangkat tidak berguna sama sekali.
  • Emergency Sweep Recovery: Pengguna dapat mengimpor frasa pemulihan (seed phrase) cadangan mereka pada dompet baru untuk memindahkan (*sweep*) seluruh dana ke alamat baru secara instan, sekaligus mendaftarkan alamat kartu lama yang hilang ke dalam daftar hitam (*blacklist*) node global.

Kondisi Termal Pembukaan Kunci (Thermal Locks & Escrow)

Untuk kebutuhan transaksi bersyarat (smart contract), Thermoledger mendukung skema penguncian koin deklaratif berbasis status termal jaringan dan otorisasi fisik:

  • Temporal Spin Lock (Time-Lock): Menangguhkan validasi transisi spin di kisi validator untuk alamat tertentu hingga tinggi blok L1 atau rentang waktu epoch tercapai (berguna untuk vesting atau tabungan berjangka).
  • Threshold Lock (Energy Bound): Koin terkunci di dalam sirkuit dan hanya dapat dicairkan jika stres termal jaringan ($\Delta H$) berada di bawah batas tertentu (menjamin transaksi diselesaikan saat biaya jaringan termurah).
  • Multi-Card Physical Signatures (Escrow): Mengunci dana di dompet L1 dan hanya melepaskannya jika menerima gabungan tanda tangan fisik (*State Energy Proof*) dari kartu penjual dan pembeli secara terotentikasi.

Core Node APIs

Validator Core Node menyediakan antarmuka API HTTP REST berkinerja tinggi untuk berinteraksi dengan sirkuit mesin konsensus.

1. POST /api/v1/transaction/prepare

Menyiapkan draf transaksi dengan menangkap status entropi jaringan paling mutakhir.

Request Body:

{
  "sender_public_key": "0x03a6bc...",
  "recipient": "0x02bf1a...",
  "amount_joule": 500000
}

Response Body:

{
  "status": "prepared",
  "entropy_salt": "0x9f8e7d...",
  "target_energy_state": "0x3c2b1a..."
}

2. POST /api/v1/transaction/commit

Mengirimkan bukti transaksi fisik yang telah ditandatangani oleh perangkat hardware/emulasi pengguna.

Request Body:

{
  "signed_energy_proof": "0x8a7b6c...",
  "transaction_payload": "0x123abc..."
}

Response Body:

{
  "status": "committed",
  "block_height": 10420,
  "settled_entropy": "0x0000a1b2..."
}

3. GET /api/v1/network/entropy-health

Mengembalikan indikator performa fisik ekuilibrium jaringan global.

Response Body:

{
  "global_entropy_stability": "99.98%",
  "active_analog_nodes": 4209,
  "global_power_draw_watts": 841.8,
  "energy_saved_vs_evm_mwh": 14.2
}

4. POST /api/v1/faucet

Mengirimkan 10 $DUIT (1.000.000.000 Joule) dari dompet Treasury L1 (0x01a2b3) ke alamat tujuan untuk kebutuhan uji coba jaringan testnet.

Request Body:

{
  "recipient": "0x03a6bc..."
}

Response Body:

{
  "status": "success",
  "tx_id": "0xfaucet9f8e7d6c",
  "amount_duit": 10.0
}

The Harvester Interface

Menyediakan jalur koneksi data tanpa latensi (*low-latency streaming interface*) bagi pengembang bot kuantitatif pihak ketiga.

Kompetisi Bot Arbitrase

Bot bersaing secara mekanis berdasarkan **Prinsip Aksi Terkecil (Least Action Principle)**:

  • Minimum Energy Path Optimization: Mengatur ulang urutan transaksi di mempool L2 agar transisi *spin* sirkuit analog seminimal mungkin. Bot yang menghasilkan "gesekan" termal paling kecil akan dieksekusi lebih awal secara alami oleh sirkuit.
  • Noise Prediction: Modul AI lokal memperkirakan fluktuasi sirkuit termal validator untuk menyelaraskan waktu injeksi transaksi dengan fluktuasi acak yang menguntungkan.
