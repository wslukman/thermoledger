## EXECUTIVE SUMMARY
Project Renungan adalah sebuah paradigma baru dalam teknologi desentralisasi yang meninggalkan komputasi digital biner tradisional ($0$ dan $1$) kaku yang boros energi. Proyek ini mengimplementasikan arsitektur Thermodynamic / Physics-as-Computation, di mana validasi konsensus dan perpindahan nilai tidak diatur oleh kode smart contract linear, melainkan oleh pencapaian kesetimbangan entropi fisik alami (State Energy Proof).

Dengan memanfaatkan koin native $DUIT yang dirancang hiper-langka (hanya 1.000.000 koin), ekosistem ini mengintegrasikan merchant retail fisik (DuitLap Cashier) dan perangkat keras independen (hardwallet analog) ke dalam satu jaringan yang diamankan langsung oleh hukum termodinamika alam.

## 1. CORE NETWORK ARCHITECTURE & CONSENSUS
### 1.1 Paradoks "Jendela Noise" & Keamanan Termal
Sistem menolak konsep keacakan semu (pseudo-random) buatan algoritma komputer digital digital biner. Sebagai gantinya, jaringan memanfaatkan thermal noise alami dari sirkuit perangkat keras sebagai bagian intrinsik dari perhitungan probabilistik.
- **One-Time Pad Alami**: Noise termal bertindak sebagai kunci kriptografi sekali pakai yang berubah setiap milidetik mengikuti fluktuasi entropi atomik perangkat keras.
- **Imunitas Peretasan**: Untuk memalsukan transaksi, peretas tidak bisa sekadar memanipulasi baris kode digital; mereka harus memanipulasi hukum fisik entropi lokal pada sirkuit validator pada milidetik yang sama. Transaksi berbahaya otomatis teredam dan diabaikan sebagai "distorsi kotor" oleh sirkuit analog.

### 1.2 Konsensus: State Energy Proof (SEP)
Menggantikan mekanisme Proof-of-Work (PoW) dan Proof-of-Stake (PoS) konvensional.
$$\Delta S_{\text{universe}} \ge 0$$
Setiap transaksi divalidasi dengan memetakan jalur komputasi analog menuju kesetimbangan energi minimum (Minimum Energy Path). Validitas dicapai ketika status jaringan lokal mencapai ekuilibrium termal yang sinkron dengan protokol global global.

### 1.3 Pemulihan Alami: Cooling-Off Period (Time-Lock Fisik)
Jika terjadi anomali atau pemutusan jaringan massal (misalnya gangguan internet global), sistem tidak memerlukan koordinasi manual manusia (hard fork).
- Node akan mendeteksi penurunan sinkronisasi getaran entropi global.
- Sistem otomatis masuk ke mode Membeku (safe mode) secara mandiri, mengunci seluruh status saldo $DUIT.
- Sistem kembali Mencair secara otomatis begitu sirkuit fisik mendeteksi stabilitas kesetimbangan energi termal jaringan telah pulih.

## 2. METODE KOMPETISI BOT ARBITRASE
Jaringan ini mengeliminasi komponen biaya gas (gas fee wars) digital murni yang diskriminatif terhadap modal. Kompetisi antar-bot (seperti evolusi algoritma bot trading) bergeser secara radikal:
- **Minimum Energy Path (Jalur Energi Terkecil)**: Validator secara mekanis memprioritaskan transaksi yang memerlukan perubahan status (state alteration) paling minimal pada sirkuit fisik perangkat keras. Bot berkompetisi menyusun struktur kode algoritma seefisien mungkin agar dieksekusi lebih awal secara alami oleh alam sirkuit chip.
- **Noise Prediction (Prediksi Kebisingan Termal)**: Bot mutakhir memanfaatkan modul AI lokal untuk mempelajari pola fluktuasi thermal noise pada pasokan sirkuit validator jaringan. Bot yang berhasil menyelaraskan waktu injeksi transaksinya dengan fluktuasi entropi alami akan mendapatkan konfirmasi instan.
- **Local Thermodynamic Ponds (Layer 2)**: Di Layer 2, transaksi mikro dikumpulkan dalam bentuk Bukti Matriks Entropi sebelum dikunci ke Layer 1. Bot berkompetisi membangun transaksi mikro yang saling menguntungkan (cooperative balance) di dalam kolam lokal tanpa risiko kehilangan biaya energi (reverted transaction tanpa biaya gas hangus).

## 3. TOKENOMICS & MODEL EKONOMI VALIDATOR
### 3.1 Profil Koin Native $DUIT
- **Total & Maksimal Suplai**: 1.000.000 $DUIT (Hiper-Langka akibat 99.9% komitmen pembakaran fase transisi).
- **Satuan Pecahan Terkecil**: Joule (J) atau Calorie (Cal), di mana:

- **Mekanisme Migrasi (Bridge)**: Pengguna token $DUIT lama di jaringan BASE (Web3 EVM) mengunci aset mereka lewat smart contract jembatan, untuk mencetak koin native $DUIT dengan rasio 1:1 di Jaringan Termodinamika Renungan.

### 3.2 Aliran Pendapatan Validator (Incentive Structure)
Validator yang mengoperasikan simpul jaringan (baik melalui emulasi perangkat lunak laptop atau perangkat keras khusus) memperoleh pendapatan dari tiga sumber:
- **Thermodynamic Block Rewards**: Koin $DUIT baru dicetak dan diberikan kepada validator yang sistem komputasinya paling efisien mencapai ekuilibrium entropi dengan konsumsi energi terendah.
- **Micro-Energy Fees**: Biaya transaksi mikro bernilai sangat rendah dibayarkan oleh pengguna sebagai kompensasi penggunaan kapasitas termal sirkuit validator. Karena biaya listrik operasional chip mendekati nol watt, margin keuntungan bersih (net profit margin) validator jauh lebih tinggi dari penambang tradisional.
- **Proof-of-Useful-Entropy (Penyewaan Otak Analog)**: Saat lalu lintas transaksi sepi, kapasitas pemrosesan probabilitas analog pada chip validator disewakan kepada pihak luar untuk melakukan simulasi sains terapan (AI probabilistik, riset molekuler, pemodelan cuaca).

## 4. SISTEM DOMPET & HARDWARE WALLET (HARDWALLET)
Wallet pada jaringan ini berevolusi menjadi Thermodynamic Vault & Generator (Kubah & Generator Termodinamika).

### 4.1 Arsitektur Perangkat Dompet
Sistem mendukung diversifikasi bentuk wallet dengan arsitektur sirkuit yang konsisten:
- **Hot Wallet (Software-Emulated)**: Aplikasi lintas platform dibangun menggunakan Flutter (Dart). Berjalan di perangkat mobile atau laptop dengan mengemulasikan kalkulasi matriks probabilitas termal melalui algoritma perangkat lunak khusus.
- **Hardwallet USB Dongle**: Perangkat fisik USB yang menampung chip analog/memristor mandiri. Laptop hanya berfungsi sebagai layar visual; seluruh proses penandatanganan kriptografi dikerjakan secara fisik di dalam sirkuit internal dongle yang terisolasi total dari internet digital.
- **Hardwallet Kartu NFC**: Kartu tipis portabel yang memanfaatkan induksi medan dekat (NFC) untuk memindahkan State Energy Proof langsung ke aplikasi kasir merchant DuitLap via smartphone.

### 4.2 Otentikasi Biometric Entropy
Sistem menghilangkan ketergantungan pada kata sandi biner teks statis yang rentan terhadap kebocoran. Transaksi ditandatangani dengan menggabungkan sidik jari atau interaksi biometrik pengguna dengan profil keacakan noise termal perangkat keras pada detik spesifik tersebut.

## 5. DUAL-LAYER AUDITING & TRANSPARANSI
### 5.1 Model Audit Pihak Ketiga (Third-Party Auditor)
Karena arsitektur sistem memotong bahasa pemrograman kaku seperti Solidity atau Rust di lapisan konsensus intinya, proses audit dibagi oleh dua keahlian independen:
- **Audit Lapisan Digital (Software Layer)**: Firma audit siber memeriksa integritas kode simulasi Python/C++, keamanan SDK Dart di aplikasi Flutter, serta proteksi dari serangan manipulasi API tradisional (Man-in-the-Middle).
- **Audit Lapisan Fisik (Hardware Layer)**: Laboratorium independen, ahli teori informasi, dan insinyur elektro mensertifikasi bahwa chip fisik/memristor menghasilkan tingkat keacakan entropi murni yang bebas dari bias statistik dan tahan terhadap intervensi fisik eksternal (Side-Channel Attacks).

### 5.2 Blockchain Explorer: DuitScan (Thermoscan)
Berperan sebagai penyedia transparansi publik yang dikembangkan menggunakan Flutter Web (berbagi basis kode komponen UI dengan aplikasi wallet utama).
- **Penyedia Infrastruktur Awal**: PT Universal Digital Property menyewa infrastruktur server awan independen yang menjalankan indexer Python/C++ untuk mengompilasi data jaringan mentah ke dalam basis data performa tinggi.
- **Dasbor Termodinamika**: Selain menampilkan riwayat transaksi koin $DUIT dan saldo merchant, halaman utama menampilkan visualisasi Entropy Health Counter dan grafik penghematan daya listrik global (Energy Efficiency Metrics).

**Request Body:**
```json
{
  "sender_public_key": "0x03a6bc...",
  "recipient": "0x02bf1a...",
  "amount_joule": 500000
}
```

**Response:**
```json
{
  "status": "prepared",
  "entropy_salt": "0x9f8e7d...",
  "target_energy_state": "0x3c2b1a..."
}
```

`POST /api/v1/transaction/commit`
Mengirimkan bukti transaksi fisik yang telah ditandatangani oleh perangkat hardware/emulasi pengguna.

**Request Body:**
```json
{
  "signed_energy_proof": "0x8a7b6c...",
  "transaction_payload": "0x123abc..."
}
```

**Response:**
```json
{
  "status": "committed",
  "block_height": 10420,
  "settled_entropy": "0x0000a1b2..."
}
```

### 6.2 Client SDK (Flutter / Dart Interface)
Pustaka komponen Dart yang diintegrasikan langsung ke dalam ekosistem aplikasi kasir retail Merchant DuitLap dan dompet digital konsumen.

```dart
import 'package:duit_thermo_sdk/duit_thermo_sdk.dart';

void initializeDuitEcosystem() async {
  // 1. Menginisialisasi koneksi ke node lokal atau jembatan hardware
  DuitWallet wallet = DuitWallet(nodeUrl: "https://node1.duitscan.io");
  
  // 2. Mendeteksi dan mengunci koneksi dengan Hardwallet Dongle USB/NFC
  bool isHardwareConnected = await wallet.connectHardware();
  
  if (isHardwareConnected) {
    print("Hardwallet Analog Terdeteksi. Keamanan dijamin Hukum Termodinamika.");
  }
}

void executeMerchantPayment(String recipientAddress, double amountInDuit) async {
  // Pecahan otomatis dikonversi ke satuan internal Joule (1 DUIT = 10^8 Joule)
  int amountInJoule = DuitConverter.toJoule(amountInDuit);
  
  try {
    // Eksekusi transaksi mikro instan melalui enkapsulasi Layer 2
    var txResult = await DuitWallet.sendMicroTransaction(
      recipient: recipientAddress,
      jouleAmount: amountInJoule,
      authMode: AuthMode.biometricEntropy
    );
    
    print("Transaksi Kasir Sukses. Terkunci di Blok: ${txResult.blockHeight}");
  } catch (e) {
    print("Gagal mencapai kesetimbangan energi transaksi: $e");
  }
}
```
